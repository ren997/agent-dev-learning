# Agent 开发学习计划

## 学习目标

通过本计划，你将从零开始理解 Agent 的基本原理，并逐步做出一个可以调用工具、保存记忆、完成任务拆解的简单 Agent。

最终目标：

- 理解 Agent 和普通聊天机器人的区别
- 掌握 Prompt、工具调用、记忆、RAG、任务规划等核心概念
- 能手写一个最小 Agent
- 能使用框架搭建更复杂的 Agent 应用
- 完成一个可展示的小项目

## 阶段一：理解 Agent 基础

预计时间：2 到 3 天

学习内容：

- 什么是 Agent
- Agent、Chatbot、Workflow 的区别
- Agent 的基本组成：LLM、Prompt、Tools、Memory、Planner、Executor
- Agent 的常见应用场景：学习助手、客服助手、代码助手、数据分析助手

练习任务：

- 用自己的话解释 Agent 是什么
- 画出一个简单 Agent 的执行流程
- 想 3 个你自己想做的 Agent 应用

## 阶段二：掌握大模型 API 调用

预计时间：3 到 5 天

学习内容：

- API Key 的使用方式
- Messages 格式：system、user、assistant
- Temperature、max_tokens 等常用参数
- 流式输出
- 错误处理和重试

练习任务：

- 写一个命令行聊天程序
- 给模型设置不同角色，观察回答变化
- 实现连续对话上下文

推荐技术：

- Python
- OpenAI SDK 或其他兼容 OpenAI 接口的 SDK

## 阶段三：手写最小 Agent

预计时间：5 到 7 天

学习内容：

- Agent 循环：观察、思考、行动、反馈
- 工具函数的设计
- 让模型决定是否调用工具
- 工具调用结果如何返回给模型

练习任务：

- 实现一个可以调用计算器工具的 Agent
- 实现一个可以保存学习笔记的 Agent
- 实现一个可以读取本地文件内容的 Agent

最小项目：

- 学习规划 Agent
- 输入学习目标后，自动拆解学习阶段、每日任务和复习计划

## 阶段四：加入记忆能力

预计时间：3 到 5 天

学习内容：

- 短期记忆：当前会话上下文
- 长期记忆：数据库或文件存储
- 用户画像：学习目标、偏好、进度
- 什么时候需要记忆，什么时候不需要记忆

练习任务：

- 用 JSON 文件保存用户学习目标
- 用 SQLite 保存学习记录
- 让 Agent 根据历史进度生成下一步任务

## 阶段五：学习 RAG

预计时间：5 到 10 天

学习内容：

- 什么是 RAG
- 文档切分
- 向量化 Embedding
- 向量数据库
- 检索结果如何交给模型回答

练习任务：

- 把几篇学习资料导入知识库
- 实现基于资料的问答
- 比较直接问模型和基于 RAG 问答的区别

推荐工具：

- Chroma
- FAISS
- LlamaIndex
- LangChain

## 阶段六：学习 Agent 框架

预计时间：7 到 14 天

学习内容：

- LangGraph
- AutoGen
- CrewAI
- LlamaIndex Agent
- 多 Agent 协作
- 状态图和工作流编排

练习任务：

- 用 LangGraph 重写之前的最小 Agent
- 做一个多角色协作 Agent，例如规划者、执行者、检查者
- 对比手写 Agent 和框架 Agent 的优缺点

## 阶段七：完成综合项目

预计时间：10 到 20 天

项目建议：个人学习助手 Agent

功能要求：

- 用户输入学习目标
- Agent 生成阶段性学习计划
- 每天生成学习任务
- 支持保存学习笔记
- 支持查询历史学习记录
- 支持基于资料库问答
- 支持总结学习进度

可选增强：

- Web 页面界面
- 文件上传
- 多用户支持
- 任务提醒
- 学习报告导出

推荐技术栈：

- Python
- FastAPI 或 Streamlit
- SQLite
- Chroma 或 FAISS
- LangGraph

## 每周安排建议

第 1 周：

- 理解 Agent 基础
- 学会调用大模型 API
- 写出命令行聊天程序

第 2 周：

- 手写最小 Agent
- 实现工具调用
- 实现学习规划 Agent 原型

第 3 周：

- 加入记忆
- 使用数据库保存学习进度
- 优化 Prompt 和工具设计

第 4 周：

- 学习 RAG
- 建立自己的学习资料库
- 实现基于资料的问答

第 5 周：

- 学习 LangGraph 或其他 Agent 框架
- 重构前面的 Agent 项目
- 增加状态管理和多步骤任务执行

第 6 周：

- 完成综合项目
- 整理代码
- 写项目说明文档
- 准备演示

## 推荐学习顺序

1. 先不用框架，手写基础版本
2. 再学习工具调用
3. 然后加入记忆
4. 再学习 RAG
5. 最后使用框架重构
