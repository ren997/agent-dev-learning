# Lesson 0001: 什么是 Agent

这一课的目标很小：你要能分清大模型应用里的 Chatbot、Workflow 和 Agent。只有先分清这三种应用形态，后面写工具调用和记忆时才不会混乱。

## 一句话定义

**Agent 是一个由大模型驱动的程序，它能根据目标选择下一步行动，并通过工具和反馈推进任务。**

这里的关键不是“大模型会聊天”，而是它进入了一个循环：观察当前状态，决定下一步，调用工具或回答，再根据结果继续。

## 大模型应用的三种常见形态

这里说的是“大模型应用”的三种形态，不是 Agent 的三种形态，也不是大模型本身的三种形态。LLM 是底层能力，Chatbot、Workflow 和 Agent 是基于 LLM 做出来的不同应用方式。

### Chatbot

主要负责回答。用户问一句，模型回一句。

```text
输入 -> 模型 -> 输出
```

### Workflow

主要负责按固定步骤执行。步骤通常由开发者提前写死。

```text
输入 -> 步骤 A -> 步骤 B -> 输出
```

### Agent

主要负责根据目标动态选择行动。它可以决定要不要调用工具、下一步查什么、何时结束。

```text
目标 -> 模型决策 -> 工具/回答 -> 反馈 -> 再决策
```

## Agent 的最小组成

- **LLM**：负责理解任务、生成计划、决定行动。
- **Prompt**：告诉模型角色、规则、输出格式和边界。
- **Tools**：模型可以请求调用的外部能力，例如函数、API、数据库、文件系统。
- **Loop**：模型行动后拿到反馈，再决定是否继续。
- **State**：保存当前任务的上下文、历史消息、工具结果或用户进度。

复习这些词时，可以打开 [Agent 开发术语速查](../reference/agent-glossary.md)。

## 一个后端开发者视角的类比

你可以把 Agent 想成一个特殊的后端服务：普通服务的分支逻辑由代码里的 `if/else` 控制，而 Agent 把一部分“选择下一步”的逻辑交给模型。

代码仍然负责边界、工具实现、数据存储、权限和错误处理。

## 小练习

### 练习 1

下面哪个更像 Agent？先自己回答，再看答案。

1. 用户输入主题，程序固定生成标题、摘要、正文。
2. 用户说“帮我整理这个项目”，程序先查看文件，再决定读哪些文件、生成计划、修改代码、运行检查。
3. 用户问“Python 的 list 是什么”，模型直接解释。

答案：第 2 个更像 Agent。因为它不是固定步骤，而是根据任务和中间反馈动态决定下一步。
