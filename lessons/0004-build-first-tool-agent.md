## 常见卡点

### 卡点 1：看不懂为什么要循环

因为 Agent 不一定一次就能结束。

它可能先请求工具，再根据工具结果继续判断下一步。

### 卡点 2：看不懂 `messages` 为什么越来越长

因为 `messages` 不只是聊天记录，它还是 Agent 当前任务状态的一部分。模型要靠它知道：

- 用户要做什么
- 自己刚才请求了什么工具
- 工具返回了什么结果

### 卡点 3：觉得这和普通函数调用差不多

差别在于“是否调用工具、调用哪个工具、传什么参数”不是你用 `if/else` 写死的，而是模型在运行时决定的。
